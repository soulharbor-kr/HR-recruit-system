## 1. 로그인

1. 시스템 주소 접속
2. 아이디 / 비밀번호 입력 후 **로그인** 클릭
3. 로그인 성공 시 지원자 목록(대시보드) 화면으로 이동

> **계정 정보는 관리자에게 문의하세요.**

---

## 2. 평가 진행 (평가위원)

### 2-1. 대시보드 확인

- 로그인 후 지원자 카드 목록이 표시됩니다.
- 각 카드에 **완료** (초록) / **대기중** (회색) 상태가 표시됩니다.
- 상단에 **"Supabase & Google Drive 연동됨"** 표시가 있어야 정상입니다.

### 2-2. 평가 시작

1. 평가할 지원자 카드의 **"평가 시작하기"** 버튼 클릭
2. 화면이 좌우로 나뉩니다:
   - **왼쪽**: 지원자 제출 서류 (PDF)
   - **오른쪽**: 평가 점수 입력 폼

### 2-3. 점수 입력

오른쪽 평가 폼에서 5개 항목을 슬라이더로 입력합니다:

| 항목 | 배점 | 설명 |
|---|---|---|
| 직무능력 | 20점 | 직무 수행에 필요한 지식과 기술 |
| 지원동기 | 20점 | 지원 동기의 진정성과 명확성 |
| 인성/품성 | 20점 | 기본적인 인성과 태도 |
| 적극성/사회성 | 20점 | 조직 융화력 및 대인관계 |
| 인상/표현력 | 20점 | 의사소통 능력 및 태도 |

- 슬라이더를 좌우로 드래그하여 점수 조정
- 우측 상단에 **실시간 총점** 표시 (100점 만점)
- 하단 **특이사항 / 종합 의견** 란에 메모 입력 가능 (선택)

### 2-4. 평가 제출

- **"평가 완료 및 제출"** 버튼 클릭
- 제출 후 대시보드로 돌아오며 카드가 **완료** 상태로 변경됩니다.
- 이미 제출한 평가는 **"평가 수정하기"** 버튼으로 재수정 가능합니다.

---

## 3. 종합 결과 확인 및 출력

### 3-1. 종합 결과 보기

1. 대시보드 우측 상단 **"종합 결과 보기"** 버튼 클릭
2. 전체 지원자 × 전체 평가위원의 점수가 표로 표시됩니다.
3. 자동으로 총점 합산 및 순위가 계산됩니다.

### 3-2. PDF로 저장 / 인쇄

1. 결과 화면 상단 **"인쇄하기 / PDF 저장"** 버튼 클릭
2. 브라우저 인쇄 창에서 **대상(Destination)** 을 **"PDF로 저장"** 선택
3. 저장 클릭

> 인쇄 결과물에는 지원자 성명, 순위, 각 평가위원 점수, 서명란이 포함됩니다.

## 4. 지원자 추가 / 변경

코드 수정 없이 Supabase에서 직접 관리합니다.

### 4-1. 지원자 PDF 추가

1. [supabase.com](https://supabase.com) 로그인 → 프로젝트 선택
2. 왼쪽 메뉴 **`Storage`** → `applicants` 버킷 클릭
3. **`Upload files`** 로 PDF 업로드
   - 파일명은 영문/숫자만 사용 (예: `06.pdf`, `07.pdf`)

### 4-2. 지원자 정보 등록

**`SQL Editor`** 에서 아래 SQL 실행:

```sql
INSERT INTO applicants (id, name, position, file_path, application_date)
VALUES ('06', '홍길동', '신입 공채', '06.pdf', '2026-03-17');
```

### 4-3. 지원자 삭제

```sql
-- 평가 데이터도 함께 삭제
DELETE FROM evaluations WHERE applicant_id = '06';
DELETE FROM applicants WHERE id = '06';
```

Storage에서도 해당 PDF 파일을 삭제합니다.

## 5. 평가위원 계정 관리

[supabase.com](https://supabase.com) → 프로젝트 → **`Table Editor`** → `evaluators` 테이블에서 직접 수정합니다.

### 계정 추가

```sql
INSERT INTO evaluators (id, name, password, role)
VALUES ('eval4', '신규위원', '비밀번호', 'evaluator');
```

### 비밀번호 변경

```sql
UPDATE evaluators SET password = '새비밀번호' WHERE id = 'eval1';
```

### 계정 삭제

```sql
DELETE FROM evaluators WHERE id = 'eval4';
```

| 컬럼 | 설명 |
|---|---|
| `id` | 로그인 아이디 |
| `name` | 화면에 표시될 이름 |
| `password` | 로그인 비밀번호 |
| `role` | `evaluator` (평가위원) 또는 `admin` (관리자) |

## 6. 문제 해결

### 로그인이 안 될 때
- 아이디/비밀번호 확인
- Supabase `evaluators` 테이블에 계정이 있는지 확인

### PDF가 안 보일 때
- Supabase Storage `applicants` 버킷의 파일 존재 여부 확인
- 버킷이 **Public** 으로 설정되어 있는지 확인
- `applicants` 테이블의 `file_path` 값이 Storage 파일명과 일치하는지 확인

### 평가 데이터가 저장되지 않을 때
- 대시보드 상단 연동 상태 확인 ("Supabase & Google Drive 연동됨")
- 주황색 "테스트 모드" 표시 시 Railway 환경변수 확인 필요 → 관리자 문의

### 종합 결과에 점수가 '-' 로 표시될 때
- 해당 평가위원이 아직 평가를 완료하지 않은 상태입니다.
